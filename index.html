Serverless Cost Optimisation - Replacing Managed Ingestion Pipelines

Cloud Cost Optimisation / FinOps
  • Replaced 10 AWS OpenSearch Ingestion Service (OSIS) pipelines on a media production platform with a serverless Lambda + DynamoDB Streams architecture, removing always-on managed-pipeline cost.
  • Built a passthrough handler that deserialises DynamoDB Stream records, derives the OpenSearch document ID from the primary key, maps stream event types to index/delete actions, and uses external document versioning for conflict-safe writes.
  • Packaged the solution as a reusable Terraform module (a map of objects driven by for_each) so pipelines can be added or removed declaratively as the platform grows.
  • Reduced ingestion infrastructure cost by over 95%, from around USD 13,900/month to around USD 120/month, while preserving throughput and indexing behaviour.
  • Technologies: AWS Lambda, DynamoDB Streams, OpenSearch, Terraform, Python, boto3

Root-Cause Investigation - Asset Ingest Race Condition

Media Systems / Reliability
  • Diagnosed a race condition in a cloud media-asset-management ingest pipeline, where an encoder's temporary-file rename pattern caused the storage gateway to register a deletion event and silently skip the finished asset.
  • Performed log forensics across dozens of rotated gateway logs, identifying approximately 4 PB of duplicated media asset data arising from multiple temp-file, atomic-save, and partial-download patterns.
  • Defined a set of scanner ignore patterns to suppress transient files and prevent false deletion events, and confirmed a reliable manual recovery step for affected assets.
  • Produced a troubleshooting runbook and a structured incident report so the diagnosis and recovery procedure are repeatable by the wider team.
  • Technologies: AWS, media asset management tooling, cloud file storage gateway, Linux, log analysis
